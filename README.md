## 💡 프로젝트 소개

### 이제 냉장고는 '정리하는' 것이 아니라 '활용하는' 것입니다

- 냉집사는 냉장고 사진 한 장으로 AI가 식재료를 즉시 인식하고, 지금 바로 만들 수 있는 맞춤형 레시피를 추천해 드립니다.
- 식재료를 일일이 입력할 필요 없이 카메라 촬영만으로 냉장고 재고를 파악하고, 유튜브 영상·쇼핑 링크까지 한 번에 제공합니다.
- 식재료 인식부터 레시피 추천, 영상 가이드, 부족 재료 구매까지 식생활 전 과정을 하나의 AI 기반 앱에 담았습니다.

### 추진 배경 및 필요성

현대 사회에서 1인 가구 및 맞벌이 가구의 증가로 인해 집에서 요리하는 빈도가 불규칙해지면서, 냉장고에 방치된 식재료가 유통기한을 넘겨 음식물 쓰레기로 발생하는 일이 지속적으로 나타나고 있다. 이는 개인의 경제적 손실로도, 환경 오염을 유발하는 것에 있어서도 개선이 필요한 문제로 인식되고 있다.

연구에 따르면 음식물 쓰레기의 상당량이 최종 유통 및 가정 내 소비 단계에서 발생하며, 특히 가정 부문에서의 발생 비중이 높은 것으로 보고된다. 이에 따라 가정 내 음식물 쓰레기 저감을 위한 체계적인 관리 방안이 필요하다.

또한, 최근 급격한 외식 물가 상승, 요리 예능 등으로 인해 직접 요리를 해 먹으려는 '홈 쿡' 수요가 늘어나고 있으나, 요리에 익숙하지 않은 초보자들은 한번 요리에 사용하고 남은 재료, 혹은 냉장고에 쌓여가는 수많은 식재료들을 어떻게 활용해야 할지 몰라 매번 새로운 식재료를 구매하거나 배달 음식에 의존하는 악순환을 겪고 있다.

따라서 사용자가 현재 보유하고 있는 재료의 활용도를 극대화하고, 누구나 쉽고 간편하게 요리에 접근할 수 있도록 돕는 지능형 보조 시스템의 필요성이 매우 높아진 상황이다.

## 🚀 주요 기능

### 📸 냉장고 식재료 인식 및 관리

- 스마트폰 카메라 촬영/업로드 시 AI가 이미지를 분석해 식재료 자동 추출
- 수동 편집(추가/수정/삭제) 기능 지원
- 사용자가 일일이 텍스트를 입력할 필요 없이, 빠르고 정확하게 냉장고 재고 현황 파악

### 🍳 맞춤형 레시피 추천 및 안내

- 인식된 식재료 기반 즉시 조리 가능한 최적 레시피 매칭
- 유튜브 링크 연동으로 신뢰도 높은 요리 영상 링크 제공
- 메뉴 선정 고민 해결 및 영상 가이드를 통한 직관적인 요리 경험

### 🔄 대체 식재료 안내 기능

- 특정 재료 부재 시, 맛과 특성이 유사한 다른 재료로 대체 제안 (예: 봄동 → 상추)
- 재료가 부족해도 유연하게 요리를 완성할 수 있어 식재료 활용 극대화

### 🛒 부족 식재료 쇼핑 연동

- 선택한 레시피 조리에 필요한 부족 재료를 파악하여 즉시 구매 외부 링크 제공
- 요리 계획부터 장보기까지 연결하여 번거로움 단축

### 🔔 맞춤형 푸시 알림

- 관심 식재료의 특가 할인 정보 발생 시 실시간 알림 발송
- 식재료 구매 지원 및 앱의 지속적인 재방문 유도

### 👤 맞춤형 프로필 설정

- 알레르기 유발 식품 제외 및 식이 제한(비건 등), 선호 요리 종류 반영 필터링
- 개인의 건강 상태와 취향을 고려한 맞춤형 식단 제공

### 🍜 남은 배달 음식 활용 레시피 추천

- 먹고 남은 배달 음식 종류를 입력/촬영하면, 냉장고 재료와 조합한 이색 재활용 요리 제안
- 처치 곤란한 남은 배달 음식을 쉽게 해결하여 음식물 쓰레기 감소

### 🌤️ 날씨 및 계절 맞춤 추천

- 계절 및 날짜 데이터와 연동하여 제철 식재료 레시피 및 트렌디한 유행 레시피 제안
- 시기에 맞는 트렌디한 메뉴 추천으로 식생활의 즐거움 증대

## 📖 사용법

### 개발 환경 설정

#### 1. 저장소 복제

```bash
git clone https://github.com/kookmin-sw/2026-capstone-30
cd capstone
```

#### 2. API 키 설정

`server/.env` 파일에 OpenRouter API 키를 입력 후에 파일 명을 .env로 수정:

```
OPENROUTER_API_KEY=발급받은_키_입력
PORT=3000
```

#### 3. 서버 실행

```bash
cd server
npm install
npm run dev
```

서버가 정상 실행되면 다음 메시지가 출력됩니다:
```
서버 실행 중: http://0.0.0.0:3000
```

#### 4. Flutter 앱 실행

서버를 켜둔 상태에서 새 터미널을 열고:

```bash
cd flutter_app
flutter pub get
flutter run --release
```

> 에뮬레이터에서 디버그 모드(`flutter run`) 실행 시 Impeller 렌더러 충돌이 발생할 수 있습니다. `--release` 옵션을 사용하세요.

```
│       ├── services/
│       │   ├── api_service.dart     # HTTP 호출 (analyzeImage / getRecipes / getRecipeDetail)
│       │   └── storage_service.dart # SharedPreferences (저장 레시피, 프로필, 재료)

│           ├── home_screen.dart              # 이미지 업로드 → 재료 분석 → 레시피 추천

    ├── server.js                    # 메인 서버 (API 라우터 포함)
    ├── config.js                    # 환경 설정
    └── package.json                 # 서버 의존성
```
